For general questions, discussions, and community help:

- **Categories**:
  - 💡 **Ideas** — Feature proposals and enhancements
  - 🙏 **Q&A** — Ask questions, get answers
  - 📣 **Announcements** — Project updates
  - 🛠️ **Development** — Technical discussions
  - 🌐 **Show and Tell** — Share your implementations

### Community Guidelines

When seeking help:

1. **Search first** — Your question may already be answered
2. **Be specific** — Include relevant details, versions, and error messages
3. **Provide context** — Code snippets, configuration files, logs
4. **Follow up** — Mark questions as resolved when answered
5. **Be respectful** — Follow our [Code of Conduct](CODE_OF_CONDUCT.md)

## Reporting Issues

### Bug Reports

Report bugs via [GitHub Issues](https://github.com/buildworksai/saraise/issues):

**Include**:
- **Title**: Clear, concise description
- **Environment**: OS, Python version, Node version, database version
- **Steps to reproduce**: Detailed, step-by-step instructions
- **Expected behavior**: What should happen
- **Actual behavior**: What actually happens
- **Logs/Screenshots**: Relevant error messages or screenshots
- **Module**: Which module is affected (CRM, Accounting, etc.)

**Template**:
```markdown
## Environment
- OS: [e.g., macOS 14.2]
- Python: [e.g., 3.11.6]
- Node: [e.g., 20.10.0]
- PostgreSQL: [e.g., 14.10]
- SARAISE version: [e.g., 0.1.0]

## Steps to Reproduce
1. Go to '...'
2. Click on '...'
3. See error

## Expected Behavior
[What should happen]

## Actual Behavior
[What actually happens]

## Logs
```
[Paste relevant logs]
```

## Additional Context
[Any other relevant information]
```

### Issue Triage

Issues are triaged based on:
- **Critical**: System down, data loss, security breach
- **High**: Major feature broken, blocking issue
- **Medium**: Minor feature broken, workaround exists
- **Low**: Cosmetic, documentation, enhancement

## Feature Requests

### Proposing Features

We welcome feature suggestions!

**Before proposing**:
1. Check existing issues and discussions
2. Review [Application Architecture](docs/architecture/application-architecture.md)
3. Ensure alignment with project goals

**How to propose**:
1. Open a **GitHub Discussion** in the "Ideas" category
2. Describe the problem and proposed solution
3. Include use cases and benefits
4. Discuss implementation approach

**Feature request template**:
```markdown
## Problem Statement
[What problem does this solve?]

## Proposed Solution
[How should it work?]

## Use Cases
1. [Use case 1]
2. [Use case 2]

## Benefits
- [Benefit 1]
- [Benefit 2]

## Implementation Notes
[Technical considerations, if any]

## Alternatives Considered
[Other approaches you've thought about]
```
